# Registro Histórico - Frontend User Registration

Este arquivo foi mantido apenas por compatibilidade histórica.

## Status

Obsoleto como fonte de planejamento ativo.

## Motivo

O conteúdo original deste documento representava um próximo passo antigo, anterior à consolidação da Onda 1. Ele já não reflete o estado atual do código nem a prioridade real do produto.

Pontos que ficaram desatualizados no documento antigo:

- tratava troca de senha como próximo incremento principal imediato
- mencionava regra de senha mínima de 6 caracteres, mas o sistema atual exige senha forte com mínimo de 8 caracteres
- não refletia convidados, reservas, histórico, estabilização de sessão nem consolidação mobile já entregues

## Fonte atual para próximos passos

Use estes arquivos no lugar deste documento:

- `docs/ROADMAP.md` -> backlog e ondas ativas
- `docs/FEATURE_MAP.md` -> o que existe hoje
- `docs/DOMAIN_RULES.md` -> regras de negócio vigentes
- `docs/ARCHITECTURE.md` -> estado técnico atual

## Observação

O comentário original sobre ausência de backend para troca de senha ficou histórico. O fluxo real foi implementado depois desta nota.
